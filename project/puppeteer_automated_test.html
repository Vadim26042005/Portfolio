Description

L'objectif est de mettre en place des tests end-to-end (E2E) avec Puppeteer et de les integrer dans une pipeline CI/CD. Si un test echoue, la mise en production est bloquee jusqu'a ce que le probleme soit resolu.

J'avais plusieurs taches consistant a creer des scripts de test Puppeteer pour trois applications web. L'objectif etait de tester l'interface utilisateur et de verifier ses fonctionnalites avant la mise en production.

Tache 1

La premiere tache consistait a creer un test Puppeteer qui verifiait la connexion dans Calljack. Calljack est une application web developpee en Golang pour accelerer la generation de documents avec des valeurs specifiques en grand nombre. Le test effectuait la connexion via l'authentification Google puis verifiait chaque champ de saisie, bouton, menu de selection et validation. Apres avoir execute tous les tests, il retournait les resultats. Mais tout n'etait pas parfait : les scripts s'executaient trop vite, j'ai donc du ajuster les timeouts pour chaque test. Malgre cela, l'application etait imparfaite et parfois les tests echouaient lors des clics sur les boutons ou pendant l'inscription. Au final, j'evaluerais l'efficacite de cette tache a environ 60% d'utilite — pas prete pour la mise en production.

Tache 2

Ma deuxieme tache etait de creer un test Puppeteer qui verifiait les fonctionnalites de paiement dans l'application appelee api-paybox. Cette application a ete creee pour la plateforme de paris DoinSport. Ici, j'ai essaye d'ameliorer mes erreurs precedentes comme les tests en dur, les timeouts en dur, la faible lisibilite et la structure lourde du code. Dans cette tache, le test de paiement devait etre execute avec differentes valeurs de cartes et differentes societes. Cela signifiait varier les numeros de carte, dates d'expiration et emetteurs. J'ai cree une liste de societes de cartes au debut de mon fichier, puis utilise ces valeurs dans les champs de saisie. Ce test etait bien meilleur et les resultats etaient globalement positifs. J'evaluerais cette tache a environ 80% d'utilite.

Tache 3

Ma troisieme tache consistait a creer un test Puppeteer qui verifiait les fonctionnalites de filtrage afin que l'application puisse etre correctement testee avant la mise en production. L'application web s'appelle DoinSport, ou mon equipe avait cree un systeme de filtrage base sur les donnees des utilisateurs. En parallele, j'ai cree un test Puppeteer qui verifiait chacun des 22 filtres. Parmi ceux-ci, il y avait 5 filtres principaux : ordre alphabetique, par numero, par nom, par date et entre deux nombres. Je n'ai pas essaye de reinventer la roue : j'ai applique une logique similaire a celle de mes taches precedentes. Cette fois-ci, cependant, j'ai ajoute des parametres pour specifier exactement quels tests executer et lesquels ignorer. Cela m'a donne de meilleurs resultats et supprime le besoin d'attendre la fin d'un test avant d'en commencer un autre. Au final, j'evaluerais cette tache a environ 90% d'utilite.
