Description

CoreFlow est un intranet pour mettre en relation les entreprises et les employes. Projet realise avec Node.js, Express, MySQL et JavaScript. Voici la documentation essentielle des routes event de l'API.

Authentification

Toutes les routes event exigent un token. Obtenez-le via POST /api/auth/login, puis ajoutez le header : Authorization: Bearer <token>. Sans token valide : 401.

Resume des routes

# Methode Route Description
1 GET /event/list/participation/:user_id/:userRole Liste des evenements par participation et role
2 GET /event/list/:id Recuperer un evenement par ID
3 POST /event/create/:userRole Creer un evenement (admin/manager)
4 PUT /event/update Mettre a jour un evenement (admin/manager)
5 POST /event/user_list_by_email Recherche utilisateurs par email

1. Liste des evenements

GET http://localhost:3000/event/list/participation/:user_id/:userRole

Params : user_id (integer), userRole (admin, manager, employe, rh). Admin recoit tous les evenements ; les autres recoivent eventLevelOne et eventLevelTwo.

2. Evenement par ID

GET http://localhost:3000/event/list/:id

Retourne { message, event }.

3. Creation d'un evenement

POST http://localhost:3000/event/create/:userRole

Champs obligatoires : titre, type_evenement, date_debut, date_fin (format YYYY-MM-DD HH:mm:ss), organisateur_id, niveau (1 = entreprise, 2 = service). Optionnels : description, lieu, inviter (emails separes par virgules), departement, est_obligatoire, nb_places_max, statut, commentaire.

{
  "titre": "Un meeting",
  "type_evenement": "meeting",
  "date_debut": "2026-06-21 12:00:00",
  "date_fin": "2026-06-21 14:00:00",
  "organisateur_id": 1,
  "niveau": "1"
}

4. Mise a jour d'un evenement

PUT http://localhost:3000/event/update

Body obligatoires : id, organisateur_id. Champs modifiables : titre, description, type_evenement, date_debut, date_fin, lieu, est_obligatoire, nb_places_max, statut, niveau, inviter, statut_participation, commentaire. Erreurs : 400, 403 (PERMISSION_ERROR), 404 (NOT_FOUND).

5. Recherche utilisateurs par email

POST http://localhost:3000/event/user_list_by_email

Body : { "email": "admin" }. Retourne { message, usersInfos } (tableau { id, email }). 404 si aucun resultat.

Niveaux et visibilite

Niveau 1 : evenement entreprise, visible par tous. Niveau 2 : evenement de service, visible selon participations. Admin : voit tout. Manager / Employe / RH : voit selon user_id et participations.
